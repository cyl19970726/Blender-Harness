# JYGC 景点动画（L2 景点合同）

## Goal

扫真实景点建筑 → 触发该景 Blender 预渲染效果层动画（~12s）→ **动画收束时浮出该景印章纹样定格 = 打卡完成·解锁印章**。一次扫描同时完成打卡+惊艳+传播。收束纪念态浮出印章纹样这一帧是**缝合契约**（内容与玩法在同一帧缝合），也是 runtime 的 checkin 确认帧；没有这一帧 = 不算过景点合同。识别成功→播动画→服务端 checkin→解锁印章的 runtime 闭环见 #135 §1。

## 1. 景点合同五要素（换个景点仍成立=这五条被违反）

景点合同 = 扫真楼时贴在真楼上的固定机位效果层。与卡面合同（手持平面物、微缩世界、虚拟运镜、扑镜满屏）**根本不同**——别把卡面语法搬进景点。

1. **固定机位**：渲染视角锁死主触发图的机位，全片一个机位。
2. **无运镜**（铁律，原因=相机属于用户）：用户手机就是相机，用户自己会走动会晃。视频里任何推拉摇移都会和真机相机运动叠加=穿帮。景点合同里"运镜"一词等于 bug。（卡面合同的虚拟运镜/扑镜在这里全部作废。）
3. **长出点**：每个效果必须从真楼的**具体部位**长出来（门洞/匾额/檐角/石缝/暗天空区…），不是浮空贴纸。浮空贴纸/贴在画面中央不咬合真楼=拒。长出点在每景创意卡里逐一定位（#135 §4）。
4. **加法光 vs SBS alpha 选型**：
   - 光效（金光/声波/鎏金/朝旭/灯河/雷）= **黑底加法层**（add blend，黑即透明，只加光不遮挡真楼）。
   - 实体（贤士光影/灯笼/剪影/题字/石狮描金）= **SBS alpha**（side-by-side，左色右 alpha）。
   - 铁律：加法光层必须黑底，不透明层禁盖死实时相机区域（盖死=离线概念，非 live-AR）。神庙类越纯加法光越安全（不塑神不显灵）。
5. **freeze**：AR 触发锁定后冻结位姿（freeze-after-lock），叙事型景点动画直接冻结防抖，见 ar-anti-jitter skill；效果层不追帧跟随，锁定即定。

## 2. 配准工艺步骤（触发图=渲染视角合同）

主触发图不是"识别用的一张图"，它是**渲染视角的合同**：Blender 相机必须还原它的机位，否则效果层贴不上真楼。步骤：

1. **触发图 = 视角合同**：选定主触发图即锁定渲染机位，全片据此渲染。
2. **fSpy / 手动 photo-match 还原相机**：从触发图反解相机内外参，导入 Blender。
3. **建筑代理白模**：按触发图建门洞/匾额/楼阁/檐角/石墙的白模块，**只作光效发射面与遮挡体，不渲染本体**（本体是真楼，用户相机里已经有）。代理白模是长出点的几何依据也是遮挡依据。
4. **composite preview = 真机模拟帧**：效果层逐帧叠回触发图（`composite_preview.py`/`.mp4`）= 真机会看到的样子。白模线框叠触发图，配准误差可见即拒。**验收看 composite preview，不看裸效果层**——裸层看不出贴没贴上真楼。

## 3. 触发图规格（三类图严格分开）

主触发图规格（#135 §5，L2 第一物理前提）：
- **夜景/黄昏蓝调固定机位优先**（暗天空 = 加法光画布；白天过曝天空无暗区 = 堵死加法光，进贤门白天图三硬伤之一）。
- 正对建筑中轴、**彻底清场**无人无遮挡（拱门内外人流会污染追踪）、无任何水印。
- 长边 ≥1920（石墙/石缝类识别命脉的 ≥2560）。
- 每景同机位加拍 1-2 张**细节特写作小触发图**（石狮头→摸狮交互、匾额特写等）。
- **三类图严格分开**（docs/architecture/ASSET_LAYOUT.md §3 铁律）：参考图（外部对标，不进产品）/ 触发图（实拍锚点，AR 识别用，落 `_assets-src/triggers/<spot>/` + PROVENANCE）/ 动画美术图（overlay 渲染产物）三者物理目录不得混放。触发图源片 PROVENANCE 必写拍摄日期/时段/机位/EXIF——缺一项配准白模就对不齐。

现状：八景暂无可用触发图。进贤门那张临时触发图（曾在 `wechat-jinxianmen/.../marker.jpg`，1024×768，带三硬伤）已随脚手架于 2026-07-08 删除（历史在 tag `h5-final`，清理记录见 git 历史 @03940cf8）；新触发图落 `wechat-gucheng` 时按 `docs/architecture/ASSET_LAYOUT.md` 归位（`markers/<spot>-marker.jpg`），源片进 `_assets-src/triggers/<spot>/`+PROVENANCE。八景触发图统一补拍是 L2 的第一物理前提（#135 §2）。

## 4. 创意方法（内容卡驱动 · history_anchor or die）

- **内容卡驱动**：史料层 [H-XX] → 寓意层 → **高潮拍 = 寓意可视化**。每 beat 必挂一个 [H-XX] 史料锚（#135 每景创意卡已给）。双保险：技术锚=天坛语法（实物先行→点亮→长出→高潮→收束纪念态），内容锚=史料 [H-XX]。
- **history_anchor or die**：判据 = "把这套创意换到另一座楼还成立吗？"。成立 = FAIL（说明它靠的是通用炫技不是这座楼的历史）。anchor 越不可换楼越好（#136 §6 排序维度）。
- **六拍骨架**（景点合同 ~12s 标准结构）：起（晓光/暮色/夜死寂落在长出点）→ 点亮（匾额/檐角逐点亮，声波起）→ 长出（金光/幻影/声波自长出点涌出）→ **高潮拍（寓意可视化，全片质量之王）** → 揭小贤入场（该景身份：引路/提灯/儒衫/合十）→ 收束（浮出印章纹样定格=缝合契约）。
- **互斥多创意 → 定帧赛马 → tournament**：新景点先出 N 个**互斥**创意（视觉主角/情绪/高潮拍型互斥，趋同则后认领者改向），**用定帧赛马不做多 fork 全渲**（定帧不过闸禁碰 runtime）；G0 共享地基先行 → N fork 并行 → tournament 择优（可杂交）→ winner 进完整生产。**进贤门 #136 是活模板**（四候选 A 叩城入贤/B 谯楼晓角/C 独存之门/D 朝旭，G0 四件套，fork 执行契约，tournament 编排，择优阈值）——做新景点直接照 #136 结构走。

## 6. A/V 合同（潮乐体系 · 声音绑动作）

- 每拍产出：画面 / edge owner / 音乐动机(motif) / 音效 / 台词 [H-XX]。八景音乐走**潮州音乐体系**，每景 motif 与年代/情绪对应：晓角号（进贤门·晓）、晚钟+庙堂乐（城隍/双峰·暮）、雅乐古琴（学宫·文）、大锣鼓唢呐（石狮桥·春节喜庆）、夯土大锣鼓+洪钟（禁城·固若金汤）、滚雷+雨后弦乐（雷神殿）、留声机+弦诗+手风琴（洋楼·乡愁）。具体每景 motif 见 #135 §4。
- **声音绑定动作铁律**（空手倒茶之戒）：每个声音必须绑一个画面动作——晓角声绑檐角点亮、晚钟绑声波荡开、串铃绑石狮迸金、纸展声绑情书展开、洪钟绑整墙鎏金。声音空响没有对应动作 = 拒。
- 候选评审必含**音画合板 review**（带声直读视频，不是看静帧）；动态必须动态审，blocking preview 用 kimi 直读视频，静帧不构成 motion 证据（#136 §5）。

## 7. runtime 接线

- runtime 底座 = `scenic-xr` 组件（`<xr-ar-tracker mode="Marker">` + 效果层视频纹理），进贤门已接线，WIP 可用。景点合同用两种混合模式：加法光层=add blend（黑底），实体层=SBS alpha。
- **XR-Frame 黑底加法混合需一次真机 spike**（#135 §7 / #136 G0-c）：SBS alpha 已验证，**加法混合未验证**——景点合同 runtime 的物理前提，真机截图为证后才能全线开工。这是 G0 共享地基的一部分（见 §8）。
- 代码只做 runtime 合成/触发/UI/音频 cue，**不生产最终视觉**；程序化/代码生成的最终视觉效果已判死（#135 §0）。
- 触发图/marker/发布视频登记入 `docs/reference/ar-assets/manifest.json`（登记才算存在）；GLB ≤2MB，超 2MB 的发布视频走 CloudBase，落点规则见 docs/architecture/ASSET_LAYOUT.md §1。

## 8. G0 共享地基（多 fork 前先行，防不可比）

多创意赛马**开 fork 前必须先做 G0 共享地基**，一次做好全 fork 继承——否则四个 fork 各配各的相机/输出契约/rubric，渲出来根本无法横向比较（不可比 = 赛马作废）。G0 四件套（#136 §2 活模板）：

- **G0-a 配准包**：fSpy/photo-match 相机 + 建筑代理白模（`<spot>-base.blend`）+ 配准精度 board（白模线框叠触发图，误差可见即拒）。
- **G0-b 输出契约**：分辨率（按触发图比例，拍板后写死）/ 24fps / ~12.5s=300 帧 / 加法层=黑底 H.264 / 实体层=SBS alpha / `composite_preview.py`。
- **G0-c XR 加法混合真机 spike**（见 §7）。
- **G0-d 评审 rubric v01 文件**（评委引用版本号，见 blender-harness skill / #131 评审契约）。

fork 只做美术层（光效/材质/动画/合成），**禁改**配准包/输出契约/rubric（发现地基 bug 提 issue 不私改）。harness 三层（感知/行动/反馈）、gate-status、check-gate-status 聚合、金反例回归见 blender-harness skill（现役以 #131 正文 + PR #134 为准，注明"PR #134 尚未合并"）。

## 9. 生产纪律速查（景点合同专属，全套见 #130/#131）

- **定帧先行**：定帧不过闸禁碰 runtime（V4 单图直接接 runtime 之败）；创意用定帧赛马，不做多 fork 全渲。
- **资产质量 = f(峰值入镜覆盖)**：高潮拍是全片质量之王，扑镜物/涌出物盖屏时的质量决定全片（Gate D v01 死于 proxy 盖屏）。景点合同里"峰值入镜"= 高潮拍那几帧长出物占屏最大时。
- **proxy 生命周期**：引入代理白模即声明退场闸（代理白模只作发射面/遮挡体，绝不能作为可见本体漏进成片——Gate C ACCEPT 的 proxy 原样漏进 Gate D 之戒）。
- **来源诚实机检**：AI 生成件/采样音源必须在 manifest 声明；pass owner 溯源资产由 .blend 遍历推导，手写声明会说谎。
- **每版一个假设**；同层连败 2 次熔断出一张定向诊断板（顶五轮盲调）；死范式熔断（2D 22版/real-model 9版之戒）。
- **观察≠奖励**：agent 自查用感知层的板，但过闸权归独立评审（自评通胀"栏杆不咬合活了6版"之戒）；metrics 只有拒绝权。
- **每景做完把新坑追加**到本 skill §5/§9 或 #135 §3（活文档）。
